Client-side form validation

An initail check and an important feature of good user experience. Alows the user to fix the incalid data cought on the client-side straight away. This should not replace exhaustive security measures. You should always perform security checks on any form-submitted data on the serer-side as well as the client-side. client-side validation is too easy to bypass.

Form Validation

A check to see if the inputed data in a form is in the correct format and within the constraints set by the applocation.

  • * This field is required
  • Please enter you phone number in the format xxx-xxxx
  • Please enter a valid email address
  • Your password needs to be between 8 and 30 characters long and contain one uppercase letter, one symbol, and a number.

    • Why validate forms?

  • We want to get the right data in the right format
  • We want to protect our users' data
  • We want to protect ourselves.

    • Types of Client-side Validation

    Form validation: uses HTML5 form validation features. Generally doesn't require much JavaScript and has a better performance. Not as customizable.

  • required: specifies whether a form field needs to be filled in before the form can be submitted.
  • minlength/maxlength: specifies the minimum and maximum length of textual data (strings)
  • min/max: specifies the minimum and maximum values of numerical input types.
  • type: specifies whether the data needs to be a number, email address, or some other specific preset type.
  • pattern: specifies a regular expression that defines a pattern the entered data needs to follow.

    • JavaScript: uses JavaScript and is completely customizable but you need to create it all.
