Ch 10: Testing and Debugging

Errors

  • System error: there's a problem with the system or external decices with which the program is interacting.
  • Programmer error: the program contains incorrect syntax or faulty logic.
  • User error: the user has entered data incorrectly, which the program is unable to use.

    • Exceptions

    An error that produces a return value that can be used by the program to deal with the error. A Stack trace also alows the program know where the error occured.

    Warnings

    Wasrnings occure if there is an error in the code but is not enough to cause the program to fail.

    'use strict'

    Strict mode produces more exceptions and warnings and prohibits the use of some deprecated features. Not using strict mode is often referred to as 'sloppy mode'.

  • To implement strict mode, simply use the string 'use strict'; on the first line in your javaScript code.

    • alert()

    Alert stops a program from runing untill OK is clicked which allows us to effectively put breakpoints in the code that allows us to see any message/variables we want.

    console

    Console provides a number of methods for logging information and debugging. However, alert() is recomended over this.

    try, catch, finally

  • try: try running on the following program
  • catch: If the program had issues running, report an error and run any listed code.
  • finally: runs code regardless of whether an exception occurred or not.
